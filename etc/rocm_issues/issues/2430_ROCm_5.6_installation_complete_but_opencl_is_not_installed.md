Hello,
I followed the quick start installation guide at https://rocm.docs.amd.com/en/latest/deploy/linux/quick_start.html
and successfully installed ROCm 5.6 on my computer (5800x + 6800xt, ubuntu 22.04).
To check the installation, `dkms status` and `/opt/rocm/bin/rocminfo` works well, and `rocm-smi` also has a correct output.

However, when checking `/opt/rocm/opencl/bin/clinfo`, it will return "no such file or directory", and when I check the /opt/rocm/ directory, there is no opencl. All dirs are as follows:
```
bin  hipblas    hipfft   hipsolver  hsa      lib      llvm  rccl        rocblas  rocm_smi  rocsolver  share
hip  hipblaslt  hiprand  hipsparse  include  libexec  oam   rocalution  rocfft   rocrand   rocsparse
```
Also, when I check `torch.cuda.is_available()`, it will return `False`.

Looking forward to your kindly suggestions.

### 评论 #1 — vitduck (2023-09-08T04:25:34Z)

Hi, 

if `rocminfo` shows correct information, then I don't think `clinfo` is strictly needed. 
But if you require it, then it is provided separately via ```rocm-clinfo``` package. 

Regarding pytorch, perhaps the runtime is not built to support `gfx1030`, you can override this behavior with 
```
$ export HSA_OVERRIDE_GFX_VERSION=10.3.0
```
You may be able to run pytorch, but there is no guaranteed of the outcome.
